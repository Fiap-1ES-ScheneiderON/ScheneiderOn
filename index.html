Dentro do projeto utilizamos conceitos de diversas matérias do 1º semestre na Engenharia de Software da FIAP.
Nosso primeiro passo foi a idealização do projeto, para isso usamos conceitos de sustentabilidade como inspiração, focamos em atender o tripé da sustentabilidade (Triple Bottol Line). Com esse objetivo em mente, nasceu a Schneider On.
Mas para dar certo tinha um coisa muito importante a ser feita, o planejamento. Para isso usamos um modelo kanban com alguns rituais do scrum aprendidos na matéria de Software Design. Planejando as entregas, ministrando prazos, priorizando atividades e alertando sobre impedimentos.
Para apresentarmos o projeto, a matéria Storytelling foi crucial para realizar uma apresentação bem elaborada, fazendo uma linha do tempo e trazendo os pontos principais do projeto, como a dor que ela resolve, objetivos e como resolver o problema.
Após criarmos um design para o projeto, com o HTML, CSS e Bootstrap 5 visto nas matérias de Front-End Design foi usado para a montagem das páginas e estilização do conteúdo! Fazendo o projeto sair do papel.
Utilizando a Lógica de programação com Java Script conseguimos animar as nossas páginas do projeto, colocando vida e deixando mais dinâmico.
